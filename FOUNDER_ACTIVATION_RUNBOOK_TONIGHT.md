## 5. Rollback Triggers (Hard Rules)

Rollback only if one of these happens:
1. Auth breaks broadly
2. Payments break broadly
3. Uploads fail consistently
4. Streams fail globally
5. Runtime survivability collapses

Do NOT rollback for:
1. Cosmetic UI defects
2. Minor layout issues
3. Non-critical warnings
4. Isolated edge-case glitches with safe recovery

## 6. Incident Ladder (If Triggered)

1. Detect: Log exact symptom, time, route, user impact
2. Triage: classify as critical or non-critical
3. Recover: apply known recovery action
4. Verify: confirm flow restored
5. Report: append incident line to recovery log

Critical incident SLA tonight:
1. First response within 5 minutes
2. Stabilization attempt within 15 minutes
3. Rollback decision by 20 minutes if unresolved
