# LEARNING_AND_EVOLUTION_SYSTEM.md
## How the Platform Gets Better Without Losing Its Soul

---

## CORE PRINCIPLE

The platform can learn, optimize, and evolve over time — but it may NEVER:
- Drift from the music-magazine identity
- Break the 1980s visual DNA
- Override the crown ethics or discovery-first rules
- Remove Big Ace override points
- Auto-publish unapproved content

Evolution = better performance, better discovery, better conversion.
NOT = identity change, style drift, structural redesign.

# FEEDBACK_LOOP_ENGINE.md
## How Behavior Signals Flow Back Into the System

---

## SIGNAL COLLECTION

Every significant user action produces a signal:
```
USER ACTION → LOG EVENT → SIGNAL QUEUE → ALGORITHM → OPTIMIZATION
```

## KEY FEEDBACK LOOPS

### Loop 1: Discovery Loop
```
Fan sees artist in Lobby Wall (position 1, 0 viewers)
    ↓ click
Fan enters room
    ↓ watches 5+ minutes
Discovery Bot registers: this artist converts at this position
    ↓
Algorithm gives artist a "conversion boost" flag
    ↓
Artist surfaces more frequently in discovery slots
```

### Loop 2: Content Loop
```
Article Feature card on Homepage 3
    ↓ click-through rate tracked
Low CTR → Algorithm flags for editorial review
    ↓ suggest new cover image or headline
High CTR → Algorithm marks article type as "proven"
    ↓ more articles of this type are prioritized
```

### Loop 3: Crown Loop
```
Crown Winner announced
    ↓ platform-wide engagement tracked for 7 days
High engagement → Crown Bot notes this artist type
    ↓ informs future featured-artist slot recommendations
Low engagement → Editorial Bot flags for issue improvement
```

# EXPERIMENT_AND_OPTIMIZATION_RULES.md
## How Safe Experiments Run

---

## EXPERIMENT TYPES

| Type | Max Users | Duration | Approval | Auto-Promote |
|---|---|---|---|---|
| Card layout test | 10% | 1 week | Algorithm | If CTR +15% |
| Headline test | 20% | 3 days | Algorithm | If CTR +10% |
| Lobby sort variant | 5% | 2 weeks | Big Ace | Never auto |
| Sponsor timing test | 10% | 1 week | Big Ace | If no CTR loss |
| Crown announcement style | 5% | 1 week | Big Ace | If engagement +10% |

## EXPERIMENT LAWS

1. Every experiment must have a clear success metric
2. Every experiment must be reversible in < 5 minutes
3. No experiment may affect the crown, ranking ethics, or discovery-first rule
4. No experiment may change the visual DNA
5. If experiment produces net negative → auto-rollback within 24h
6. All experiments are logged and visible to Marcel and Big Ace
7. No user is permanently opted into an experiment without consent

# QUALITY_CORRECTION_ENGINE.md
## How the Platform Fixes Itself

---

## AUTO-CORRECTION TRIGGERS

| Condition | Detection | Auto-Fix | Escalate? |
|---|---|---|---|
| Broken card (no data) | Card renders empty | Show fallback state | Log only |
| Dead route | 404 detected | Show 404 page + suggest | Log + weekly review |
| Stale issue (unpublished) | Issue date > 8 days | Show previous issue | Alert Big Ace |
| Expired sponsor | Campaign end date passed | Swap to house ad | Log |
| Orphan profile (no posts) | 180 days no activity | No action (never suppress) | Log |
| Ghost avatar | Session ended, avatar stuck | Janitor Bot removes | Log |
| Bot failure | Bot didn't run in schedule | Fallback mode + retry | Alert Big Ace |
| Crown calculation error | Tie or error in formula | Hold current crown | Alert Big Ace immediately |
| Lobby sort broken | Same artist top 3 days running | Force re-sort | Log |

# HUMAN_OVERRIDE_AND_APPROVAL_CHAIN.md
## When a Human Must Intervene

---

## BIG ACE ONLY (Never automated)

- Crown rule changes
- Platform emergency lock
- Sponsor approval
- Bot start/stop
- Feature flag changes
- Ranking freezes
- Emergency broadcasts
- Content removal (legal)
- Artist verification

## REQUIRES BIG ACE REVIEW (Algorithm flags, human decides)

- Crown calculation tie-breaker
- Experiment promotion to permanent
- New bot deployment
- Major layout changes
- Issue theme approval
- Any change to discovery-first rules

- Card layout optimization
- Article recommendation order
- Notification timing
- Lobby refresh rate
- Performance scaling
