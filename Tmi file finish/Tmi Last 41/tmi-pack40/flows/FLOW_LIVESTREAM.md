# EVENT FLOW: ARTIST GOES LIVE
## Start to Finish — Every System Involved

```
TRIGGER: Artist clicks "Go Live" in /dashboard/artist

STEP 1 — API: POST /api/livestream/start
  → Creates Livestream record (status: "live", generates streamKey)
  → Creates Room record (roomType: LIVE_STAGE, isLive: true, scene: "live-stage")
  → Returns: { streamKey, ingestUrl, roomId, playbackUrl }

STEP 2 — Artist OBS/Streaming Software
  → Artist pushes RTMP stream to ingestUrl using streamKey
  → Media server receives stream, starts HLS transcoding
  → Generates playbackUrl (m3u8 manifest with 5 quality variants)

STEP 3 — BROADCAST ENGINE
  → broadcast.engine detects stream healthy
  → Sets StreamStatus → "live"
  → Activates broadcaster personality (if configured)
  → Starts lower-thirds rotation

STEP 4 — REALTIME WebSocket
  → Emits to /rooms namespace: { type: "host_started", roomId }
  → Lobby wall receives event → re-sorts rooms ORDER BY viewer_count ASC
  → (0 viewers = position 1 — Platform Law #1)

STEP 5 — DISCOVERY: 0 viewers = position 1
  → Artists at 0 viewers ALWAYS appear first
  → This is non-negotiable — never modify this sort
  → As viewers join, room moves down in sort (more viewers = less discovery priority)

STEP 6 — FAN JOINS ROOM
  → Fan clicks room on lobby wall
  → GET /api/rooms/:roomId → room details, playbackUrl
  → Frontend loads HLS player with playbackUrl
  → POST /api/rooms/:id/join → creates RoomMember record
  → WebSocket: viewer_joined event → viewerCount++
  → Lobby wall receives viewer_count event → re-sorts

STEP 7 — PARTICIPATION POINTS
  → PointsLedger: join_event +10 (on room join)
  → PointsLedger: stay_10_minutes +15 (timer fires after 10min)
  → PointsLedger: stay_30_minutes +25 (timer fires after 30min)
  → Daily cap enforced: MAX_DAILY 500 pts

STEP 8 — SPONSOR AD SLOT
  → Every 30min: POST /api/ads/slot/LIVE_ROOM_OVERLAY → always 200
  → house-ad-fallback.bot fills if no active campaign
  → Broadcast engine triggers ad_break_start event
  → Advertiser tracking: POST /api/ads/impressions

STEP 9 — TIP FLOW
  → Fan clicks tip button → POST /api/wallet/tip
  → Transaction created: type TIP, amountCents
  → Artist wallet.balanceCents += (amount - platform fee)
  → WebSocket: tip event → broadcaster shoutout triggers
  → PointsLedger: tip_artist +15 for tipper

STEP 10 — STREAM ENDS
  → Artist stops OBS → stream ingest disconnects
  → fail-safe.bot detects dropout → retries for 60s
  → After 60s: switches to FAIL_SAFE_CONTENT (standby loop)
  → If artist confirms end: POST /api/livestream/end
  → Triggers replay pipeline: raw recording → HLS VOD
  → Room closedAt set, isLive: false
  → Replay available at /replays/[id] within 10-30min

STEP 11 — ARCHIVE
  → archivist.bot takes snapshot of session stats
  → EarningsRecord created for session
  → Highlights generated by clip-highlight.bot
  → Article drafted by editorial-assembly.bot ("Live Recap: [Artist] Set")
  → Article must link to artist station (Platform Law #9)
```

**Engines involved:** realtime, broadcast, media-pipeline, scoring (points), economy (tips), ui-hud (scene), audio
**DB models:** Livestream, Room, RoomMember, PointsLedger, Transaction, Wallet, Placement, Upload, Article
**WebSocket:** /rooms, /notifications, /crown, /hype, /ads
