# @tmi/autonomy-engine

Autonomous operations and orchestration layer for the BerntoutGlobal XXL universe.

## Purpose

Coordinate the entire platform to run itself safely, intelligently, and profitably through guardrailed automation.

## Responsibilities

- Show scheduling and orchestration
- Venue rotation management
- Engagement pacing control
- Monetization pacing
- Audience flow management
- Automation bot coordination
- Experimentation management
- Learning signal collection
- Moderation escalation
- Revenue protection
- Experience freshness control

## Inputs

- Platform signals (attendance, watch duration, tips, etc.)
- User preferences and history
- Venue fatigue metrics
- Revenue metrics
- Engagement metrics

## Outputs

- Automated scheduling decisions
- Venue rotation recommendations
- Monetization pacing adjustments
- Engagement protection actions
- Revenue protection triggers

## Connected Modules

- scene-engine
- script-engine
- event-bus
- state-engine
- platform-kernel

## Connected Engines

- reward-engine
- inventory-engine

## Not Responsible For

- Direct user interactions
- Content delivery
- Real-time rendering
- Database operations

## Governance

This engine operates under strict guardrails. See:
- data/autonomy/guardrails.json
- data/bots/head-sentinel.json
