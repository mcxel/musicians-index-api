# CAST_ANALYTICS_AND_SCORING.md
## Performance Metrics for Every Cast Member

---

## PURPOSE
Measures the effectiveness of every host, Julius, and VEX interaction.
Feeds into the memory system to drive improvement.

---

## METRICS PER HOST

### Engagement Metrics
| Metric | Description | Target |
|---|---|---|
| Crowd reaction strength | Peak reaction level during segment | >7/10 |
| Contestant engagement | How long/deeply contestant responds | >30 seconds |
| Response quality | Dialogue relevance + tone | >8/10 |
| Dialogue freshness | Repetition rate of scripts | <15% repeat |
| Host popularity by event | Fan preference per event type | Tracked |

### Performance Metrics
| Metric | Description |
|---|---|
| Intro effectiveness | Did the crowd respond within 5s? |
| Follow-up success | Did follow-up deepen the moment? |
| Transition smoothness | Did the show continue without dead air? |
| Outro quality | Did the segment end well? |

---

## METRICS FOR VEX

| Metric | Description |
|---|---|
| Costume click rate | Did audience click VEX outfit to store? |
| Removal comedy score | Did crowd react positively to removal? |
| Audio timing accuracy | Was warning sound properly proximity-synced? |
| Repeat outfit flag | Did same outfit appear too recently? |

---

## METRICS FOR JULIUS

| Metric | Description |
|---|---|
| Assist effectiveness | Did user engage after Julius assist? |
| Annoyance rate | Did user dismiss or ignore Julius? |
| Action freshness | Repeat rate of last 10 actions |
| Context accuracy | Was Julius in the right mode for the page? |

---

## SCORING SYSTEM

Each cast member gets a rolling performance score (0–100):
- Updated after every event
- Displayed in admin cast analytics panel
- Flags for improvement when below 70
- Triggers memory update when above 80

---

## REPORTING

Analytics available in:
- Admin Cast Analytics Dashboard
- Per-host event breakdown
- Weekly cast performance summary
- Outfit/wardrobe performance log (VEX)
- Julius interaction log
