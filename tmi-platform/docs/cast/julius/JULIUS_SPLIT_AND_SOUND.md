# JULIUS_SPLIT_SYSTEM.md
## Julius Echo Clone Division System

---

## PURPOSE
Julius can split into multiple copies of himself for hype, comedy, prop deployment,
and performance moments. Inspired by PlayStation AR bot energy — fast, fun, surprising.

---

## SPLIT TRIGGER

Triggered by:
- Double-tap / flick-up input on Julius action button
- Major platform milestone event (win, level-up, discovery)
- Show host signals Julius for crowd moment
- Specific performance mode sequence

---

## SPLIT TYPES

| Type | Copies | Duration | Purpose |
|---|---|---|---|
| Echo Pair | 2 copies | 5–8 seconds | Quick hype beat |
| Burst | 3 copies | 4–6 seconds | Celebration moment |
| Lineup | 3–4 copies | 8–12 seconds | Show intro/reveal |
| Swirl | 3 copies, rotating | 5 seconds | Transition or magic |
| Scoreboard Crew | 2 copies | As needed | Board deploy assist |
| Shoulder Copies | 2 mini copies | 6–10 seconds | Comedy shoulder-devil style |
| Hologram | 1–2 translucent | 5–8 seconds | Info display mode |

---

## CLONE BEHAVIOR RULES

1. Clones cannot act independently of main Julius for more than 10 seconds
2. Clones do not make main sounds (only soft echo chirps)
3. Clones always perform synchronized or call-and-response actions
4. No more than 4 total Julius entities visible at once (performance budget)
5. Clones have 60-second cooldown after despawn

---

## MERGE-BACK SYSTEM

After split, Julius reassembles via one of:
- **Fold-in**: Clones fold into main body sequentially
- **Sparkle dissolve**: Clones dissolve into particles that merge into main
- **Jump-stack**: Clones jump into main body with pop sound
- **Prop capture**: Clones holding props pass them to main and vanish

---

## ONE-BUTTON ACTION SYSTEM

Single Julius Action Button controls all interactions:

| Input | Action |
|---|---|
| Tap | Quick personality action (wave, point, tip) |
| Hold | Open Julius tool wheel |
| Double tap | Split-pop burst |
| Swipe up | Lookout stretch / reveal prop |
| Swipe down | Crouch / hide / crawl |
| Swipe left | Dismiss / slide left |
| Swipe right | Present / reveal right |
| Rapid taps | Excitement chain |
| Flick | Directional split or prop launch |

---

## COOL RULE
Julius should feel like he's choosing to act — not responding to a button.
Inputs trigger probability pools, not guaranteed same action every time.

## SOUND IDENTITY
Julius communicates through sounds, not full words.
His voice is a mix of: real meerkat + chipmunk chuckle + playful chirp + personality character.

---

|---|---|---|

| Sniff | Curiosity sniff | Every 15–25s |
| Low grunt | Lookout stance | On stretch action |
| Tail flick sound | Soft swoosh | On tail animation |
| Breath | Subtle breath cycle | Always |

## FREQUENCY RULES
- Never play same sound within 10 seconds of last use
- Max 3 sounds per 20-second window
- Big personality sounds have 45-second cooldown
- Idle sounds are background-level, never startling
- All sounds muted by default in reader/article mode
