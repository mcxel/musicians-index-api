# CYPHER_VENUE_SYSTEM.md
## Cyphers Inside Venue Environments

---

## CONCEPT
A cypher is a circle-format live performance event.
The stage becomes a circle. Audience wraps around it.
Every seat has a direct view of the performer in the center.

---

## CYPHER STAGE LAYOUT

```
         [ CROWN / TITLE ]
    ╔══════════════════════╗
    ║  BALCONY (overview)  ║
    ║   ┌────────────┐    ║
    ║   │            │    ║
    ║   │  AUDIENCE  │    ║
    ║   │  (ring)    │    ║
    ║   │ ┌────────┐ │    ║
    ║   │ │CYPHER  │ │    ║
    ║   │ │CIRCLE  │ │    ║
    ║   │ │(stage) │ │    ║
    ║   │ └────────┘ │    ║
    ║   │            │    ║
    ║   └────────────┘    ║
    ╚══════════════════════╝
```

**Circle stage rules:**
- Artists enter and exit from breaks in the ring
- One performer at a time in center (or multiple for collaboration cyphers)
- Camera orbits the circle
- No blocked views — all seats wrap around

---

## VENUE OPTIONS FOR CYPHERS

Any venue can host a cypher with layout adaptation:

| Venue | Cypher Adaptation |
|---|---|
| Basement | Small tight circle, closest crowd, most intense |
| Night Club | Floor cleared, dance/rap energy |
| Amphitheater | Large outer ring, major event feel |
| Cypher Arena | Dedicated cypher venue, optimized layout |
| Warehouse | Industrial feel, raw energy |
| Bar | Intimate circle, small crowd |

---

## CYPHER LIVE STREAM MODE
Cypher can stream live to non-seated audience:
- Non-seated viewers see a camera feed from ring-view
- Seated audience in venue sees the real circle
- Both can react and vote

# BATTLE_VENUE_SYSTEM.md
## Battles Inside Venue Environments

---

## CONCEPT
A battle splits the stage into two zones.
Each competitor owns one zone.
Audience sits facing both zones.
Judges (hosted or crowd-voted) decide the winner.

---

## BATTLE STAGE LAYOUT

```
     [ JUDGE PANEL / SCOREBOARD ]
╔════════════════════════════════╗
║         AUDIENCE               ║
║   ┌──────────┬──────────┐     ║
║   │  ZONE A  │  ZONE B  │     ║
║   │          │          │     ║
║   │ Artist 1 │ Artist 2 │     ║
║   │          │          │     ║
║   └──────────┴──────────┘     ║
║    HOST ZONE (front center)    ║
╚════════════════════════════════╝
```

---

## BATTLE TYPES + VENUE MATCH

| Battle Type | Best Venues | Stage Style |
|---|---|---|
| Rapper vs Rapper | Basement, Club, Arena | Dual mic zones |
| Singer vs Singer | Concert Hall, Amphitheater | Dual spotlight |
| Dancer vs Dancer | Club, Arena, Warehouse | Wide open dual zones |
| Drummer vs Drummer | Warehouse, Arena | Drum kit dual setup |
| Comedian vs Comedian | Bar, Club, Arena | Comedy mic dual |
| Pianist vs Pianist | Concert Hall, Lounge | Grand piano dual |

---

## CROWD VOTING IN BATTLES
Audience members vote in real time:
- Julius deploys live vote board
- Crowd energy meter splits (left/right)
- Each side pulses with its artist's hype
- Winner announced when meter tips decisively

# LIVE_STREAM_VENUE_SYSTEM.md
## Going Live and Live Streams in Venue Environments

---

## CONCEPT
When an artist goes live, audience members can JOIN a venue to watch together.
Instead of watching alone, they sit in a room with other fans.
The artist appears on the main screen (or as avatar on stage).

---

## GOING LIVE FLOW

```
ARTIST clicks "Go Live"
    ↓
ARTIST selects:
  - stream type (screen, avatar, hybrid)
  - venue (if they have one unlocked)
  - audience capacity
    ↓
VENUE LOADS for audience
    ↓
AUDIENCE enters venue, takes seats
    ↓
LIVE BEGINS
    ↓
ARTIST appears on stage/screen
    ↓
AUDIENCE reacts in real-time
    ↓
ARTIST sees crowd, reactions, chat
    ↓
STREAM ENDS → outro sequence → audience release
```

---

## VENUE SELECTION FOR LIVE STREAMS

| Artist Tier | Available Venues |
|---|---|
| Free | Basement, Bar |
| Bronze | + Warehouse, Club |
| Gold | + Concert Hall, Amphitheater |
| Platinum | + Arena, Cypher Arena |
| Diamond | + World Stage (global broadcast) |

---

## GOING LIVE AUDIENCE CAPACITY

| Venue | Max Live Viewers |
|---|---|
| Basement | 50 |
| Bar | 100 |
| Warehouse | 300 |
| Night Club | 500 |
| Concert Hall | 1,000 |
| Amphitheater | 5,000 |
| Arena | 25,000 |
| World Stage | Unlimited |

**Overflow system**: If venue is full, late arrivals join an overflow room (same stream, different venue shell).

---

## ARTIST CONTROLS DURING LIVE

While live, artist can:
- See crowd (animated audience)
- See viewer count per venue
- Read reaction feed
- Trigger Julius interaction (polls, tips)
- Change camera angle
- Spotlight a specific viewer
- Trigger sound/light effect
