# SURFACE TYPE SYSTEM
## BerntoutGlobal XXL — TMI Platform
## Effective: 2026-03-23 | Status: LOCKED

---

## PURPOSE

Define the platform’s canonical surface categories and prevent visual/function drift.

---

## SURFACE TYPES

### A) Belt Surfaces
Primary: homepage/lobby discovery
Examples:
- Live now belt
- Battles/cypher belt
- Artist belt
- Magazine belt
- Sponsor belt

Characteristics:
- Horizontal rails
- Fast scan cards
- Live indicators
- Quick actions

### B) Magazine Surfaces
Primary: editorial narrative
Examples:
- Article spread
- Issue view
- Recap stories

Characteristics:
- Spread structure
- Feature typography
- Structured content zones

### C) Room Surfaces
Primary: real-time interaction
Examples:
- Arena room
- Cypher room
- Mini cypher
- Watch room
- Producer room

Characteristics:
- Main stage area
- Queue + roster
- Reactions + chat
- A/V mixer controls

### D) Dashboard Surfaces
Primary: operation/control
Examples:
- Admin command
- Booking dashboard
- Sponsor operations
- Venue operator tools

Characteristics:
- Control tiles
- Metrics panels
- Logs/event stream
- Action controls

### E) HUD Surfaces
Primary: cross-platform runtime overlay
Examples:
- Now playing
- Notifications
- Points
- Live/queue indicators

Characteristics:
- Persistent visibility
- Compact controls
- Fast context switching

---

## SURFACE-TO-ENGINE REQUIREMENTS

Every surface must map to at least one engine and one provider.

If a surface has no mapped engine/provider, it is an orphaned UI shell.

---

## COMPLETION RULE

A surface type is complete only when:
- Visual style matches PDF authority
- Wiring chain is complete
- HUD interactions are declared where applicable
- Logging/proof path exists
